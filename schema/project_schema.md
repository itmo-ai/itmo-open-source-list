# Схема карточки проекта

Каждый проект описывается отдельным JSON-файлом в папке `projects/`. Имя файла должно совпадать с `id` проекта (например, `fedot.json`).

## Обязательные поля

- **`id`** (string)  
  Уникальный идентификатор проекта латиницей (slug). Используется в URL и имени файла.

- **`title`** (string)  
  Название проекта (как правило, на английском, без перевода).

- **`description`** (object)  
  Описание проекта.
  - **`ru`** (string, **обязательно**) — описание на русском языке (1–3 предложения).
  - `en` (string, *опционально*) — описание на английском.

- **`github_url`** (string, URL)  
  Ссылка на основной репозиторий (GitHub/GitLab). Отсюда скрипт будет брать статистику (звёзды, дату обновления, лицензию, язык).

- **`lab`** (string)  
  ID лаборатории, к которой относится проект. Должен совпадать с `id` одного из файлов в папке `labs/`.

- **`area`** (string)  
  Основная тематика проекта (например: `AI for Science`, `ML Tools`, `Education`, `Robotics`).

## Опциональные поля

- **`topics`** (array of strings)  
  Список тегов. *Если не указано, скрипт автоматически загрузит их из репозитория GitHub.*

- **`links`** (object)  
  Ссылки на ресурсы:
  - `website`
  - `docs`
  - `demo`
  - `telegram`

- **`contacts`** (object)  
  Контактная информация:
  - `email`
  - `name` (имя контактного лица)

- **`maintainers`** (array of objects)  
  Список основных разработчиков/лидов: `[{ "name": "...", "github": "..." }]`.

- **`logo`** (string, URL)  
  Ссылка на логотип проекта.

- **`citation`** (object)  
  Информация для цитирования (для научных проектов).

- **`status`** (string)  
  Статус проекта: `active` (по умолчанию), `archived`, `maintenance`.
